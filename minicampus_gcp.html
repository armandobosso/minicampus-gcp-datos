1) ¿Qué es GCP y por qué usarlo?

Google Cloud Platform (GCP) es la plataforma en la nube de Google que integra servicios para almacenar, procesar y analizar datos a gran escala, con seguridad, escalabilidad y colaboración nativa entre herramientas.

Ventajas clave

  • Escalabilidad automática
  • Colaboración integrada
  • Seguridad y control de accesos
  • Pago por uso

Servicios más usados

BigQuery
Vertex AI Workbench
Cloud Storage

Casos típicos

  1. Exploración y análisis de grandes volúmenes de datos
  2. Entrenamiento y despliegue de modelos de ML
  3. Integración de fuentes internas con datasets en la nube

3) BigQuery – Data Warehouse en la nube

BigQuery ejecuta Standard SQL sobre grandes volúmenes de datos sin administrar servidores. Modelo de costos por almacenamiento y bytes procesados.

Estructura de datos

Proyecto → Dataset → Tabla

ElementoEjemplo
Proyectomi-empresa-analitica
Datasetventas
Tablaventas_2024

Consulta ejemplo

SELECT provincia, SUM(importe) AS total
FROM `mi_dataset.ventas`
WHERE fecha BETWEEN '2024-01-01' AND '2024-12-31'
GROUP BY provincia
ORDER BY total DESC;

Consejo: usa EXPLAIN y previsualización de bytes para estimar costo.

4) Vertex AI Workbench, entornos y Git

Vertex AI Workbench permite crear notebooks Jupyter en la nube con librerías de Python preinstaladas. Cada notebook corre en una instancia gestionada. Podés aislar dependencias con virtual environments y versionar con Git.

Crear entorno virtual

python3 -m venv .venv
source .venv/bin/activate
pip install pandas numpy scikit-learn
python -m ipykernel install --user --name mi_entorno

Flujo básico con Git

git clone https://github.com/tu-org/tu-repo.git
cd tu-repo
git add .
git commit -m "Actualizo notebook"
git push

5) Buenas prácticas mínimas

  • Organizá proyectos por dominio: datasets y scripts con nombres claros.
  • Evitá subir datos sensibles a repositorios públicos.
  • Documentá dependencias en requirements.txt.
  • Definí roles y permisos con el principio de menor privilegio.
